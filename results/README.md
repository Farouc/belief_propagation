# Results Guide

This guide explains how to read outputs in `results/`.

## Segmentation Results (`results/segmentation/...`)

Each dataset folder (`bsds500`, `grabcut`) contains:

- `metrics.csv`: per-image metrics
- `summary.json`: dataset-level aggregates
- `plots/`: aggregate visualizations

### `metrics.csv`
Per image:
- `iou`: Intersection-over-Union (higher is better)
- `accuracy`: pixel accuracy (higher is better)
- `runtime_sec`: runtime per sample
- `iterations`: BP iterations used
- `converged`: 1 if tolerance reached, else 0
- `fg_ratio`: predicted foreground fraction

### `summary.json`
Aggregate metrics:
- `mean_iou`, `std_iou`
- `mean_accuracy`, `std_accuracy`
- `mean_runtime_sec`
- `mean_iterations`
- `convergence_rate`
- run config (`max_iters`, `tol`, `damping`, etc.)

### `plots/`
Expected aggregate plots:
- `best_median_worst.png`: qualitative panel
- `iou_hist.png`: IoU distribution
- `accuracy_hist.png`: accuracy distribution
- `convergence_curves.png`: message-delta curves vs iterations

## Sudoku Results (`results/sudoku`)

- `metrics.csv`: per-puzzle metrics
- `summary.json`: aggregate statistics
- `plots/`: convergence/runtime/solve statistics

### Sudoku `metrics.csv`
Per puzzle:
- `solved`: 1 if valid final Sudoku and clues preserved
- `clues_consistent`: 1 if original clues unchanged
- `converged`: 1 if BP tolerance reached
- `iterations`
- `runtime_sec`

### Sudoku `summary.json`
- `solve_rate`, `solve_rate_std`
- `clue_consistency_rate`
- `convergence_rate`
- `mean_iterations`, `std_iterations`
- `mean_runtime_sec`, `std_runtime_sec`

### Sudoku `plots/`
- `convergence.png`
- `iterations_hist.png`
- `runtime_hist.png`
- `solve_rate.png`

## Clean Commit Policy

For push-ready commits, we keep concise aggregate outputs and remove bulky per-sample dumps (for example: `predictions/` folders or large `examples/` directories). These can always be regenerated by rerunning experiments.
